Issue #1 — Critical

Form submissions are broken

Backend API (Laravel on Railway) is down — all form submissions fail CriticalBackendForms
When someone fills out the Writer, Director, or Producer form and clicks Submit, the frontend calls POST /api/public/forms/[type] on the Laravel backend hosted on Railway. That server is either down, sleeping (Railway free tier spins down after inactivity), or misconfigured. The result is the "Cannot connect to server" error you saw in the screenshot.

Every submission attempt silently fails. No data is being saved. If anyone submitted a form recently, that data is gone.
Fix Options — Pick One Option A (Fast, 2 hrs): Email fallback via Resend — Replace the API call with a direct email send using Resend (free tier, no server needed). Every submission goes straight to contact@ravokstudios.com as a formatted email. Data is immediately in your inbox. No Railway dependency.

Option B (Proper, 4-6 hrs): Fix Railway deployment — Debug why Railway is failing, fix environment variables, ensure the Laravel backend is healthy, add a health check. More robust long-term but requires backend debugging.

Recommendation: Option A first, then Option B later. Get submissions working today with email, then migrate to the database backend when it's stable.

Issues #2–5 — High Priority

Form UX needs a rebuild

The forms are structurally there but the experience doesn't match what you're asking for. These are people submitting serious professional work — the form needs to feel that way.

No required vs optional field distinction HighFormsDesign
Every field looks identical — no asterisk, no visual weight difference, no label color. Currently the code says "Name and email are required, all others optional." But per your Google Doc, that's wrong — many more fields should be required or at minimum strongly encouraged. Users have no idea what they need to fill in before submitting.
Fix Add gold asterisk (*) to required fields. Add "(optional)" label in muted text to optional ones. Mark truly important fields as required in the validation logic, not just name + email.
Rating questions are plain text boxes — need 1–5 and 1–10 selectors HighFormsDesign
These questions currently render as open textareas:
Writer: "Narrative arc: How strong is this aspect?" / "Character development: How strong?" / "Story structure: How strong?"
Director: "Technical Fluency (1–10)" / "Narrative Integrity (1–10)"
Producer: "Discipline with paperwork, payroll, and reporting (1–10)"

People type random things. You can't analyze or compare that data. They need interactive selectors — the gold dot/ball style you described.
Fix Build a RatingScale component. For 1–5: five clickable circles that fill gold on select. For 1–10: ten smaller dots in a row. Selected value stores as a number. Clean, fast to implement with Tailwind.
Director + Producer forms use shorthand labels, not full questions HighForms
The Writer form uses the full question text ("Can you share the logline in 30 words or less?"). But the Director form has labels like just "Budget Experience" or "Technical Fluency" with no actual question. Same for Producer. Users don't know what to write. Your Google Doc has the full question wording — it's not being used.
Fix Replace all shorthand labels in directorQuestions[] and producerQuestions[] arrays with the full question text from your Google Doc. 30-minute task for Claude Code.
Insights page shows "Loading insights…" — content never renders HighBackendContent
The /insights page is live but broken. It's stuck in a loading state because it's trying to fetch articles from the Laravel backend — which is down. Anyone visiting the Insights page sees a spinning loader forever. This is a public-facing page tied to your brand credibility.
Fix Add a graceful fallback: if the API fails, show a clean empty state ("Insights coming soon — follow us on Instagram for the latest.") instead of a broken loading spinner. Later, when the backend is fixed, content will auto-populate.

Issues #6–11 — Medium Priority

Experience improvements

Form T&C is a wall of small text — needs a better UX MediumForms
The Terms & Conditions text is dumped as a scrollable textarea that people scroll past. The AI data processing clause in particular (Section 7.2) is legally significant. Better UX: collapsible summary with a "Read full terms" expand, and a clearly styled checkbox that confirms agreement.
Fix Split into a short summary + expandable full text. Style the checkbox to be prominent (gold border, clear label).
Producer form has 29 fields with no section grouping — high abandonment risk MediumForms
29 questions presented as one continuous scroll. No sections, no visual breathing room. Industry research consistently shows forms longer than ~10 visible questions have significantly higher abandonment. The Producer form needs section headers: "About the Project", "Your Background", "Financial Details", "Strategic Fit".
Fix Add gold section dividers + headers grouping related questions. Does not change the questions themselves, just organizes them visually.
About Us team photos not loading MediumContent
The three team member photo slots (Amanda, Thibault, Lois) have an image load fallback showing the first letter of the name. The actual photos either aren't uploaded to /public/images/team/ or the filenames don't match. Looks unfinished for a pitch-quality site.
Fix Upload correct photos to the right path, or confirm filenames match what's in the code: amanda.jpg, thibault.jpg, lois.png.
No success / error feedback after form submission attempt MediumForms
When the API is down, the toast error just says "Submission failed" with no guidance. When it works, the success just redirects to /form. Users deserve: a clear confirmation screen, what happens next (response time), and a way to follow up.
Fix Design a proper success state ("Your submission has been received. We review all submissions within 2-4 weeks.") and better error messages with a retry option.
Contact page has three "inquiry types" pointing to the same email MediumContent
General Inquiries, Partnership Inquiries, and Investor Relations all show contact@ravokstudios.com. This creates the appearance of organizational structure without actually having it. Either differentiate the emails, or simplify to one clean contact line.
Fix Either set up email routing (e.g., investors@ravokstudios.com → forwards to Amanda) or remove the fake differentiation and use one honest contact line.
No "Submit" link in the main navigation MediumDesign
The submission forms are your primary conversion tool for writers, directors, and producers. They're buried at /form/writer etc., reachable only from the Contact Us page. The Navbar has no direct path to submission. For your target audience — creators — this should be a top-level nav item.
Fix Add "SUBMIT" to the Navbar, linking to /form (the form index page showing all three types).

The site has two parts: the Next.js frontend on Vercel (always on, auto-deploys) and a Laravel backend on Railway (needs active management). The backend is where form submissions are stored and where articles live. Right now it's unreachable.

  • All static pages
  • Navigation & routing
  • Auth UI (login/register pages render)
  • Investor portal UI

✗ Not Working

  • Form submission API
  • Insights / articles API
  • Confession wall API
  • Admin data fetching
  • Email notifications
